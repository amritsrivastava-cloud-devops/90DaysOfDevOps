# üêß Linux Administration ‚Äì L2 / Senior / Principal Reference

This document reflects commands and practices expected from a **Senior or Principal Linux Administrator**, focused on **production systems, troubleshooting, performance, and reliability**.

---

## üß¨ System Identity, Boot & Kernel

| Command | Explanation |
|------|------------|
| `uname -a` | Kernel, architecture, and OS details |
| `uname -r` | Kernel version (critical for driver/module issues) |
| `hostnamectl` | View/set hostname and OS metadata |
| `uptime` | System runtime and load averages |
| `last reboot` | Reboot history |
| `who -b` | Last system boot time |
| `lsmod` | Loaded kernel modules |
| `modprobe module` | Load kernel module |
| `sysctl -a` | View kernel parameters |
| `sysctl -w key=value` | Tune kernel parameters at runtime |
| `dmesg -T` | Kernel ring buffer with timestamps |

---

## üöÄ systemd & Service Lifecycle (Deep Knowledge)

| Command | Explanation |
|------|------------|
| `systemctl status service` | Service health, logs, exit codes |
| `systemctl cat service` | View full unit file (incl. overrides) |
| `systemctl show service` | Runtime properties |
| `systemctl list-dependencies service` | Dependency chain |
| `systemctl list-units --failed` | Failed services |
| `systemctl daemon-reexec` | Reload systemd safely |
| `systemctl edit service` | Create drop-in overrides |
| `journalctl -u service -p err` | Error-level logs only |
| `journalctl --since "1 hour ago"` | Time-based log analysis |

üëâ **Interview signal:** Knowing `cat`, `show`, and `edit` separates seniors from juniors.
